# 提示词大师 Pro / PromptOps Studio v3 产品方案

日期：2026-06-09

## 1. 产品定位

提示词大师 Pro 是一个面向个人、专业从业者和团队的 Agent 驱动 PromptOps 工作台。

它帮助用户把已有提示词、业务经验、参考文档、工具上下文和专业流程，整理成可复用的 AI 资产与能力包，并支持持续优化、运行验证、反馈迭代和市场分发。

一句话定位：

> 用户提供提示词或一句话目标，Agent 帮他优化、拆解、封装、组合、验证，并沉淀成可复用 AI 能力。

产品不只是一个 Prompt 模板库，也不是普通 AI 聊天工具，而是一个让用户把提示词、文档、经验、工具和工作流封装成可复用 AI 能力的系统。

核心原则：

> 用户说人话，Agent 做结构化；用户轻量修改，系统负责沉淀复用。

## 2. 核心用户

### 2.1 普通用户

普通用户已经有一些提示词，但提示词不够稳定、不够专业、不够清晰。他们希望系统能帮自己把已有提示词优化成更高质量版本。

典型需求：

- 优化一段自己写的 Prompt。
- 根据参考资料补强提示词。
- 让输出格式更稳定。
- 让模型更懂任务边界。
- 反复迭代直到满意。

### 2.2 专业从业者

老师、律师、程序员、运营、咨询顾问、研究人员等，希望把自己的工作经验和 AI 使用方法沉淀成可复用资产。

典型需求：

- 老师沉淀出题 Skill、批改 Workflow、评分 Evaluator。
- 律师沉淀合同审查 Prompt、风险 Policy、法规 Reference。
- 程序员沉淀代码审查 Skill、GitHub MCP 上下文、测试 Workflow。
- 运营沉淀小红书模板、品牌语气 Policy、爆款分析 Workflow。

### 2.3 团队

团队希望沉淀内部最佳实践、SOP、审核规则、交付模板和新人可复用能力。

典型需求：

- 把内部 SOP 变成可调用 Workflow。
- 把团队经验变成能力包。
- 让新人直接复用成熟工作方法。
- 管理团队级资产和能力边界。

### 2.4 创作者与专家

创作者或专家希望制作资产包或能力包，上传到市场，让别人下载使用。

典型需求：

- 发布自己的 Prompt 模板。
- 发布专业 Skill。
- 发布完整能力包。
- 通过下载量、评分、评论形成影响力。

## 3. 产品核心闭环

```text
用户粘贴提示词
-> 系统理解提示词意图
-> 推荐资产 / 能力包
-> 整合参考文件和资产
-> 输出优化提示词
-> 用户编辑并多轮迭代
-> 满意后沉淀成资产
-> 多个资产组合成能力包
-> 能力包可一键使用和导出
-> 资产 / 能力包可上传市场
-> 用户可从市场下载别人的能力
-> 运行实验室验证效果
-> 反馈洞察持续改进
```

这里有一个关键产品设定：

> 工作台里用户输入的是自己的提示词内容，而不是需求表单。

系统要从提示词本身理解它想完成什么、缺什么、该插入哪些资产、该怎么改得更专业。

## 4. 最终产品结构

```text
工作台
资产库
资产构建室
能力包
市场
运行实验室
反馈洞察
设置
```

## 5. 模块一：工作台

工作台是产品第一入口，核心任务是优化用户已有提示词。

### 5.1 用户输入

用户可以在工作台中：

- 粘贴自己的提示词内容。
- 上传参考文件。
- 选择已有资产。
- 选择能力包。
- 什么都不选，直接优化。
- 让 Agent 自动推荐资产或能力包。

### 5.2 系统分析

系统需要自动完成：

- 理解提示词目标。
- 判断使用场景。
- 识别目标用户。
- 识别输入材料。
- 识别期望输出。
- 识别风险和边界。
- 识别提示词结构缺口。
- 推荐可用资产或能力包。

### 5.3 资产整合

系统可以整合：

- Prompt
- Skill
- Reference
- Policy
- Workflow
- Evaluator
- MCP 上下文
- SDK 上下文
- Tool 上下文
- Template
- Dataset
- Benchmark

注意：MCP、SDK、Tool 默认作为工程上下文参与提示词优化，不代表真实执行。

### 5.4 提示词优化输出

系统输出优化后的提示词，要求：

- 结构清晰。
- 任务明确。
- 输入输出契约稳定。
- 约束和边界明确。
- 结合用户所选资产。
- 保留用户原始意图。
- 说明本次优化使用了哪些资产和方向。

### 5.5 多版本迭代

版本迭代是工作台核心能力：

- 初始优化生成 V1。
- 用户可以手动编辑 V1。
- 系统基于用户编辑继续优化为 V2。
- V2 可以继续编辑再生成 V3、V4。
- 每轮都要保留用户编辑意图。
- 支持版本历史。
- 支持版本对比。
- 支持回退版本。
- 支持复制结果。
- 支持保存结果。

### 5.6 结果沉淀

用户满意后，可以一键沉淀为：

- Prompt 资产
- Template 资产
- Skill 草稿
- Evaluator 草稿
- Workflow 草稿
- 能力包中的某个资产

工作台解决的问题：

> 我已经有一个提示词，帮我把它优化到专业、稳定、可复用。

## 6. 模块二：资产库

资产库是所有 AI 能力原材料的管理中心。

### 6.1 资产类型

资产库管理以下类型：

- Prompt
- Skill
- MCP
- SDK
- Workflow
- Reference
- Agent
- Tool
- Evaluator
- Policy
- Template
- Dataset
- Memory
- Parser
- Benchmark
- Connector

### 6.2 资产管理能力

资产库支持：

- 新建资产。
- 编辑资产。
- 删除资产。
- 搜索资产。
- 按类型筛选。
- 按能力状态筛选。
- 按标签筛选。
- 按质量分筛选。
- 按使用次数筛选。
- 按最近使用筛选。
- 插入工作台。
- 加入能力包。
- 导入资产。
- 导出资产。
- 下载资产。
- 本地 JSON state 持久化。

### 6.3 外部链接导入

资产库支持通过外部链接导入：

- GitHub 仓库
- 文档链接
- API 文档
- MCP 文档
- SDK 文档
- 博客教程
- 公开 Prompt 集合
- Notion 文档，后续扩展
- 飞书文档，后续扩展
- 语雀文档，后续扩展

导入流程：

```text
外部链接
-> Agent 读取和摘要
-> 判断可提炼资产类型
-> 生成资产草稿
-> 用户确认
-> 保存到资产库
```

### 6.4 文件上传导入

资产库支持通过文件上传导入：

- Markdown
- TXT
- JSON
- Word
- Excel
- PDF，后续支持
- 压缩包，后续支持

导入流程：

```text
上传文件
-> 抽取文本
-> Agent 分析内容
-> 判断适合提炼的资产类型
-> 生成资产草稿
-> 用户确认
-> 保存到资产库
```

### 6.5 能力状态

资产能力状态分为：

- 仅上下文：只参与提示词上下文。
- 可编译：字段完整，可参与提示词编译。
- 可测试：可做连接测试或格式测试。
- 已连接：运行环境已配置连接。
- 可执行：用户确认后允许真实执行。

资产库解决的问题：

> 我有哪些可复用 AI 资产，它们能不能被搜索、组合、迁移和持续改进。

## 7. 模块三：资产构建室

资产构建室是产品第二核心。

它不是单纯表单，而是支持手动构建和 Agent 构建的资产生成空间。

### 7.1 手动构建模式

手动构建适合专业用户。

用户选择资产类型，然后编辑结构化字段。

不同资产类型有不同结构：

#### Skill

- 触发条件
- 输入
- 输出
- 执行流程
- 能力边界
- 验证方式
- 使用说明

#### MCP

- Server 信息
- Tools
- Resources
- Prompts
- 输入输出 schema
- 权限
- 安全边界
- 错误处理

#### SDK

- 包名
- 语言
- 安装方式
- 初始化方式
- 认证方式
- 核心方法
- 参数和返回
- 错误处理
- 测试方式

#### Workflow

- 目标
- 阶段
- 输入
- 输出
- 质量门
- 失败处理
- 最终交付物

#### Evaluator

- 评估对象
- 评分维度
- 通过标准
- 失败案例
- 输出格式
- 人工 / AI / 混合评估模式

#### Policy

- 适用领域
- 规则
- 触发条件
- 禁止事项
- 升级策略
- 拒答或降级风格
- 示例

#### Reference

- 来源
- 版本
- 范围
- 关键事实
- 术语
- 引用规则
- 限制
- 时效

### 7.2 Agent 构建模式

Agent 构建适合大多数用户。

用户只需要说一句话，或上传文档、粘贴链接：

```text
帮我根据这份教材做一个出题 Skill。
帮我把这个 API 文档整理成 MCP 资产。
帮我把这套公司 SOP 变成 Workflow。
帮我给这个提示词设计一个 Evaluator。
帮我从这个 GitHub 仓库提炼可用资产。
```

Agent 自动完成：

- 判断资产类型。
- 提炼标题。
- 总结用途。
- 拆解输入输出。
- 补充边界约束。
- 生成示例。
- 生成使用说明。
- 生成验证标准。
- 给出缺失项提醒。

用户只需要微调和确认。

### 7.3 从文档提炼资产

文档不是单独作为一个知识库摆放，而是作为资产构建室的原料。

典型流程：

```text
上传文档 / 粘贴链接
-> Agent 读取
-> 摘要内容
-> 提炼可复用能力
-> 生成资产草稿
-> 用户确认
-> 保存到资产库
```

典型例子：

- 从教材提炼出题 Skill。
- 从公司 SOP 提炼 Workflow。
- 从法规文档提炼 Reference 和 Policy。
- 从接口文档提炼 MCP 或 SDK 资产。
- 从提示词集合整理 Prompt 模板库。

### 7.4 缺失资产补齐

当能力包里缺某个资产时，用户可以点击“让 Agent 创建”。

流程：

```text
能力包编辑器
-> 发现缺失资产
-> 跳转资产构建室
-> Agent 生成资产草稿
-> 用户确认保存
-> 自动回填能力包
```

资产构建室解决的问题：

> 用户不需要懂 Skill、MCP、Workflow 怎么写，只要能描述目标，Agent 就能搭出资产草稿。

## 8. 模块四：能力包

能力包是资产的组合层。

能力包不是单个资产，而是一组围绕具体场景协同工作的资产。

示例：

```text
合同审查能力包
= 合同审查 Prompt
+ 风险 Policy
+ 法规 Reference
+ 审查 Workflow
+ 条款改写 Skill
+ 质量 Evaluator
```

### 8.1 Agent 创建能力包

用户创建能力包时，不需要填写复杂表单。

用户只需要说：

```text
我想做一个合同审查能力包。
我想做一个作文批改能力包。
我想做一个代码审查能力包。
我想做一个小红书运营能力包。
```

Agent 自动整理：

- 能力包名称
- 使用场景
- 典型输入
- 期望输出
- 推荐资产槽位
- 已有可用资产
- 缺失资产
- 使用说明

这些字段由 Agent 先自动填好，用户只做轻量修改。

### 8.2 能力包编辑器

能力包编辑器支持：

- 从资产库任意挑选资产。
- Agent 推荐已有资产。
- 查看每个资产在能力包中的角色。
- 手动调整资产槽位。
- 缺什么就跳转资产构建室新建。
- 新建完成后自动回填能力包。
- 查看完整度和质量分。

资产槽位包括：

- 核心 Prompt
- Skill
- Workflow
- Reference
- Policy
- Evaluator
- MCP / SDK / Tool
- Template
- Dataset / Benchmark

### 8.3 一键使用

能力包支持一键使用。

流程：

```text
用户选择能力包
-> 跳转工作台
-> 自动带入能力包资产
-> 用户粘贴具体提示词
-> 系统结合能力包优化提示词
```

### 8.4 导入导出

能力包支持：

- 导入
- 导出
- 下载
- 分享
- 上传市场
- 版本管理
- 使用次数记录

导出分为：

- 完整导出：包含能力包元信息、资产内容快照、资产关系和使用说明。
- 引用导出：只包含能力包结构和资产 ID，适合团队内部同步。

### 8.5 能力包质量检查

系统可以判断能力包是否完整。

质量状态包括：

- 草稿
- 可用
- 推荐
- 待补全
- 高风险缺失

示例：

- 有 Prompt：基础可用。
- 有 Reference：上下文更可靠。
- 有 Policy：风险边界更明确。
- 有 Workflow：处理流程更稳定。
- 有 Evaluator：结果可评估。
- 有 Benchmark：可做回归测试。

能力包解决的问题：

> 用户不只是保存资产，而是把一类工作封装成可反复调用的 AI 工作方案。

## 9. 模块五：市场

市场是产品的分发和增长模块。

### 9.1 市场内容

市场支持上传和下载：

- 单个 Prompt
- Skill
- MCP 资产
- SDK 资产
- Workflow
- Evaluator
- Policy
- Reference
- Template
- 完整能力包

### 9.2 市场分类

市场可以按场景分类：

- 教育
- 法律
- 编程
- 运营
- 写作
- 办公
- 研究
- 产品
- 销售
- 通用效率

### 9.3 市场展示信息

市场资产或能力包应展示：

- 名称
- 简介
- 类型
- 作者
- 使用场景
- 包含资产
- 能力状态
- 下载量
- 评分
- 更新时间
- 安全说明

### 9.4 下载到本地

用户可以：

- 下载资产到自己的资产库。
- 下载能力包到自己的能力包列表。
- 下载后继续编辑。
- 下载后加入工作台使用。

### 9.5 上传到市场

用户可以：

- 上传自己的资产。
- 上传自己的能力包。
- 设置简介和标签。
- 设置公开范围，后续支持。
- 更新已发布版本，后续支持。

### 9.6 市场安全边界

市场下载的资产需要默认安全策略：

- 默认只作为上下文资产。
- MCP / SDK / Tool 不自动变成可执行。
- 需要用户确认后才可启用连接或执行状态。
- 明确展示来源和权限风险。
- 对可执行能力进行额外风险提示。

市场解决的问题：

> 用户不仅能自己沉淀能力，还能复用别人的能力，甚至分享自己的专业能力。

## 10. 模块六：运行实验室

运行实验室用于验证提示词、资产和能力包是否真的有效。

### 10.1 对比能力

运行实验室支持：

- 不插资产 vs 插资产对比。
- 单资产对比。
- 能力包对比。
- 编译模式对比。
- 测试输入。
- 输出差异查看。
- 运行记录保存。

### 10.2 编译模式

编译模式包括：

- 可读版
- 严格版
- 工具适配版
- Agent 版
- 评估版

### 10.3 无模型 Key 模式

无模型 key 时：

- 明确显示“仅编译预览”。
- 不伪装真实运行。
- 可以查看提示词差异。
- 可以保存编译记录。

### 10.4 有模型 Key 模式

有模型 key 时：

- 可真实调用模型。
- 可保存输出。
- 可结合 Evaluator 自动评分。
- 可生成 Benchmark 回归记录。

运行实验室解决的问题：

> 资产和能力包不是摆设，它们要能证明自己让结果变好了。

## 11. 模块七：反馈洞察

反馈洞察让系统越用越聪明。

### 11.1 反馈事件

系统记录用户行为：

- 复制结果
- 编辑结果
- 删除片段
- 追加要求
- 重新生成
- 保存为资产
- 加入能力包
- 放弃版本

这些行为会被转成 FeedbackEvent。

### 11.2 Agent 反馈分析

Agent 分析反馈，判断：

- Prompt 是否表达不清。
- Reference 是否缺资料。
- Policy 是否不完整。
- Workflow 是否步骤不稳。
- Evaluator 是否标准不够。
- 能力包是否缺资产。

### 11.3 AssetPatch

系统根据反馈生成 AssetPatch。

用户可以：

- 接受补丁。
- 拒绝补丁。
- 稍后处理。

接受后：

- 更新资产版本。
- 更新能力包质量。
- 记录补丁来源。
- 保留修改原因。

反馈洞察解决的问题：

> 每一次使用，都在反向训练用户自己的资产库和能力包。

## 12. 模块八：设置与运行边界

设置页负责可信和安全。

### 12.1 基础配置

设置页包括：

- 模型 API key 配置。
- 后端 API 状态。
- 本地 JSON state 状态。
- 市场连接状态。
- MCP 状态。
- SDK 状态。
- Tool 状态。
- Connector 状态。
- 环境变量说明。
- 数据导入导出设置。

### 12.2 能力状态

能力状态分为：

- 仅上下文
- 可编译
- 可测试
- 已连接
- 可执行

### 12.3 可信原则

关键原则：

- 不把没连上的工具伪装成能执行。
- 市场下载的资产默认不具备执行权限。
- 任何 MCP / SDK / Tool 执行都需要明确配置和用户确认。
- API key 和 token 只能从环境变量或用户配置读取，不能写死在代码里。
- 高风险操作需要用户显式确认。

设置解决的问题：

> 产品必须可信，不能把上下文资产包装成已经真实执行的能力。

## 16. MVP 优先级建议

最小可用版本应优先完成：

1. 工作台提示词优化与多版本迭代。
2. 资产库管理与导入导出。
3. 资产构建室的手动构建和 Agent 构建。

6. 反馈洞察的基础事件和 AssetPatch。
7. 设置页的能力状态和运行边界。

市场可以先做本地模拟版本：

- 本地市场数据。
- 下载到资产库。
- 下载到能力包。
- 上传到本地市场。

等本地闭环稳定后，再接真实远程市场。

## 17. 产品差异化

提示词大师 Pro 的差异化不在于“存很多 Prompt”，而在于：

- 它理解用户已有提示词。
- 它可以整合资产优化提示词。
- 它能把好结果沉淀成资产。
- 它能把多个资产组合成能力包。
- 它能通过运行实验室验证效果。
- 它能通过反馈洞察持续迭代。
- 它能通过市场分享和下载专业能力。

最终愿景：

> 让每个人都能把自己的专业经验封装成可复用 AI 能力。
